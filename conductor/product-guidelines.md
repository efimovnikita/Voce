# Product Guidelines: Mistral Speaker

## 1. Voice and Tone
- **Tone:** **Minimalist**. The application's communication should be clear, functional, and direct. Avoid unnecessary jargon or conversational filler.
- **Language:** Prioritize short, actionable labels (e.g., "Play," "Settings," "Save") and concise status updates (e.g., "Loading audio...", "Playback ready").

## 2. Visual Language & Branding
- **Aesthetic:** Clean and modern, with a strong focus on the audio player's controls and visual feedback.
- **Color Palette:** **Vibrant Accents**. Use bold, high-energy colors for primary action buttons (Play/Pause) and animations (Waveform, Progress Bar) to create a lively and engaging experience against a neutral background.
- **Feedback Elements:** Visual feedback like waveform animations should be prominent and visually distinct.

## 3. Core UX Principles
- **Speed & Responsiveness:**
  - Prioritize fast startup and immediate feedback on all user interactions.
  - Optimize the generation and delivery of audio chunks to minimize latency during playback.
- **Seamless Integration:** The "Share" menu integration must be as frictionless as possible, allowing for one-tap text-to-speech conversion.

## 4. Error Handling & Recovery
- **User-Centric Recovery:**
  - When an error occurs (e.g., a network failure or an invalid API key), provide clear, actionable messages that explain the issue and suggest a resolution.
  - Avoid cryptic error codes; instead, use plain language (e.g., "Please check your Mistral API key in Settings").
  - Offer a simple way to retry the operation where appropriate.

## 5. Accessibility & Inclusivity
- **Clarity:** Ensure high contrast between text/icons and their backgrounds.
- **Feedback:** Provide visual and, where possible, haptic feedback for key player actions to assist users in various environments.
